WHY A GRAPH?

CFD mesh is already a physical graph.

CT voxel은 격자지만 CFD solver의 계산 domain은 대개 불규칙한 tetrahedral mesh다. node는 위치와 물리 state를, cell/edge는 이웃 관계를 표현한다. GNN은 이 구조를 억지로 image grid로 resample하지 않고 직접 처리한다.

THE LEARNING PIPELINE

  1. Geometry and boundary conditions

    surface/volume mesh, inlet/outlet, wall label, inflow waveform을 확보한다. 좌표계·unit·boundary definition을 field registry에 고정한다.

  2. Graph construction

    pos[N,3], tetra connectivity, derived edges, node type/wall mask, velocity history를 만든다. 원본 HDF5/XDMF는 immutable하게 보존한다.

  3. Encode–process–decode

    encoder MLP가 node/edge feature를 latent로 바꾸고, message passing processor가 이웃 간 정보를 여러 round 전달한 뒤 decoder가 다음 velocity를 예측한다.

  4. Autoregressive rollout

    예측한 u(t+1)을 다음 입력으로 다시 사용한다. 한 step accuracy와 장기 rollout accuracy는 별도의 문제다.

  5. Physics-aware training and evaluation

    data loss 외에 continuity/mass conservation 등 물리 제약을 평가한다. boundary, wall/interior, bulge/neck을 분리해 오류를 보고한다.

CORE CONCEPTS

What the model actually sees

Node features

position, velocity history, wall/inlet/outlet indicator, inflow context. 모든 feature는 source와 physical unit을 명시한다.

Edge features

relative displacement, distance, 혹은 mesh adjacency. edge가 많을수록 receptive field는 넓지만 memory와 oversmoothing 위험도 커진다.

Message passing

각 node는 이웃 message를 aggregate하고 latent state를 갱신한다. 15 rounds는 15-hop local propagation이지 전역 physics 보장을 뜻하지 않는다.

Loss

vector velocity error 외에 divergence/flux 같은 residual을 더할 수 있다. constraint 항의 scale과 weighting은 ablation으로 검증한다.

Derived WSS / OSI

velocity prediction 뒤 gradient·wall geometry로부터 계산한다. source CFD와 같은 discretization/unit인지 검증하지 않으면 임상 biomarker 비교가 성립하지 않는다.

Uncertainty

OOD geometry, inflow shift, mesh density shift를 별도 failure mode로 둔다. point estimate만으로 clinical applicability를 주장하지 않는다.

# Evaluation must report

Ordering rule: baseline MGN을 먼저 같은 data/split에서 재현하고, inflow feature, physics loss, masked pretraining은 한 번에 하나씩 추가한다. 성능 향상은 각 ablation의 동일한 rollout horizon에서만 비교한다.
